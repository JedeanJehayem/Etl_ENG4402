Consolidação e Padronização de Bases de Prestadores📝 DescriçãoEste projeto realiza o processo de transformação e consolidação de três bases de prestadores de saúde em um único arquivo CSV padronizado. O foco principal é a qualidade, consistência e preparação dos dados para análises posteriores ou integração em novos sistemas.O script unifica dados de diferentes tipos de prestadores:Hospitais SADT (Serviços de Auxílio Diagnóstico e Terapia) Hospitais Dia 🚀 Fluxo de ProcessamentoO script segue uma ordem lógica de tratamento para garantir a integridade da base final:Leitura dos arquivos CSV originais.Padronização de colunas e tratamento de valores nulos.Limpeza Textual (Remoção de acentos e normalização).Tratamento de Identificadores (CNPJ e CNES).Validação de UF e formatação de datas.Consolidação, remoção de duplicados e exportação.🛠️ Principais Transformações1. Padronização de Colunas e NulosNomenclatura: As colunas passam por remoção de acentos, conversão para maiúsculas e substituição de sublinhados por espaços para garantir compatibilidade entre datasets.Valores Nulos: Diferentes representações de ausência de dados (como NULL, NAN, N/A, -) são convertidas para um padrão único.2. Tratamento de Campos CríticosCNPJ: Limpeza de caracteres não numéricos e preenchimento com zeros à esquerda até 14 dígitos.CNES: Limpeza e preenchimento com zeros até 7 dígitos. Caso o valor exceda 7 dígitos, o registro é mantido e marcado na coluna OBSERVACAO como -MAIOR QUE 7-.UF: Validação para garantir duas letras maiúsculas correspondentes a uma unidade federativa válida do Brasil.Data de Atualização: Conversão para o formato padrão YYYY-MM-DD.3. RastreabilidadeCada registro recebe uma coluna ORIGEM (HOSPITAIS, SADT ou HOSPITAIS_DIA), permitindo identificar a fonte original do dado após a unificação.📂 Estrutura de ArquivosEntradaO script busca os seguintes arquivos no diretório de execução:Hospital - 100%.CSVsadt_100.CSVhospitais_dia_100.CSVSaídaGera o arquivo consolidado:Arquivo: base_consolidada_fator_qualidade.csvColunas Finais: NOME FANTASIA, CNES, CNPJ, UF, MUNICIPIO, DATA DE ATUALIZACAO, ORIGEM, OBSERVACAO.Formatação: Separador ;, codificação utf-8-sig e aspas em todos os campos para compatibilidade com Excel.💻 Requisitos e ExecuçãoPré-requisitosPython 3.x Biblioteca pandas InstalaçãoBashpip install pandas
Como ExecutarColoque os arquivos de entrada na mesma pasta do script.Execute o comando:Bashpython nome_do_script.py
🔍 Observações TécnicasLeitura Segura: Os arquivos são lidos inicialmente como texto (dtype=str) para evitar que o Python altere automaticamente números longos (como CNPJ) para notação científica.Compatibilidade Excel: O script aplica um prefixo nos campos CNES e CNPJ para forçar a interpretação como texto em planilhas eletrônicas.Unicidade: A remoção de duplicidade é realizada com base na combinação única dos campos CNES e CNPJ.[!TIP]Melhorias Futuras: O roadmap do projeto prevê a criação de relatórios de inconsistências separados, logs detalhados e validação cruzada entre Município e UF.
